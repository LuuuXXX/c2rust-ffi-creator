## 差分测试分级说明

本计划采用三级差分测试体系（Tier 0 / 1 / 2），分级依据为接口复杂度、风险信号数量和现有测试覆盖率。

### Tier 0 — 仅复用现有 C 测试

**定义**：不额外增加差分测试；将现有 C 测试套件链接到 Rust 实现，直接验证等价性。

**适用条件（满足全部）**：
- 接口为纯函数或状态简单的句柄接口
- 无回调函数指针
- 输出确定性强（无浮点精度依赖、无时间戳等非确定因素）
- 现有 C 测试覆盖率 ≥ 70%（关键路径）
- 无已知历史 bug 或边界行为怪癖

**验证策略**：
```
C 测试 → 链接 Rust 静态库 → 运行 → 全部通过 ✓
```

---

### Tier 1 — Golden File 对比测试

**定义**：在 Tier 0 基础上，额外增加输入/输出 golden file 对比测试，覆盖多返回路径和复杂输出结构。

**适用条件（满足任一）**：
- 接口有复杂的输出结构体（多字段、嵌套、含 padding）
- 有多个错误返回路径（每个路径需独立验证）
- 输出涉及字符串或 buffer 内容（需逐字节验证）
- 现有 C 测试覆盖率 < 70% 但 > 40%

**验证策略**：
```
标准输入集合 → C 实现 → 记录 golden 输出
标准输入集合 → Rust 实现 → 对比 golden 输出
差异报告 → 人工审查 → 确认或修复
```

**Golden File 存放路径**：`tests/golden/`

---

### Tier 2 — 模糊测试 + 差分对比

**定义**：在 Tier 1 基础上，增加模糊测试驱动的差分对比（C 实现与 Rust 实现并行运行，实时对比输出）。

**适用条件（满足任一）**：
- 接口有密集回调或复杂回调时序
- 接口依赖全局状态或有复杂初始化要求
- 历史上有较多 bug 或边界行为问题
- 接口处理不可信输入（解析器、协议处理等）
- 现有 C 测试覆盖率 < 40%

**验证策略**：
```
随机/变异输入 → C 实现（运行于 ASAN/UBSAN） → 输出 A
同一输入       → Rust 实现                   → 输出 B
A ≠ B → 记录差异，人工分类（预期差异 / 需修复）
```

**推荐工具**：
- `libFuzzer` / `AFL++` 生成输入
- `cargo-fuzz` 在 Rust 侧运行
- 差分框架：自定义 harness 或 `difftastic`
